# 数据库/数据表

## 库表操作

- [`addColumn`](../by-name/a/addColumn.md) — 给数据表添加新的列。若对分布式表、维度表或流数据表添加新列，不能使用 SQL update 语句，只能使用 `addColumn` 命令。
- [`addFunctionView`](../by-name/a/addFunctionView.md) — 函数视图是封装了访问数据库以及相关计算语句的一种特殊的自定义函数。函数视图提供了一种灵活的方式来控制用户访问数据库和表。用户即使不具备读写数据库原始数据的权限，也可通过执行函数视图，间接访问数据库，得到所需计算结果。例如，用户不能查看个别患者的名称和年龄等原始数据，但是可以获取患者的总数与平均年龄。
- [`addRangePartitions`](../by-name/a/addRangePartitions.md) — 给数据库增加新的分区。目标数据库必须是 RANGE 分区类型或 COMPO 分区类型并且至少其中一层分区为 RANGE 类型。注意，只能在最后一个现有数据分区后面添加分区，不能在第一个现有数据分区前面添加分区。
- [`addValuePartitions`](../by-name/a/addValuePartitions.md) — 给数据库增加新的分区。目标数据库必须是 VALUE 分区类型或 COMPO 分区类型并且至少其中一层分区为 VALUE 类型。
- [`backup`](../by-name/b/backup.md) — 以分区为单位，备份分布式表的数据。返回一个整数，表示备份成功的分区数量。该函数必须要用户登录后才能执行。
- [`backupDB`](../by-name/b/backupDB.md) — 备份数据库到指定路径。
- [`backupTable`](../by-name/b/backupTable.md) — 备份数据库中的表到指定路径。
- [`cacheDS!`](../by-name/c/cacheDS_.md) — 函数 `cacheDS!` 会在下次执行，并缓存数据。
- [`cacheDSNow`](../by-name/c/cacheDSNow.md) — 函数 `cacheDSNow` 立即执行并缓存数据源和缓存行的总数。
- [`cachedTable`](../by-name/c/cachedTable.md) — 创建一种特殊类型的内存表：缓存表。如果查询缓存表的时间与上次数据更新时间相距等于或超过 *retentionSeconds* 秒，会自动执行 *updateFunc* 以更新缓存表。
- [`checkBackup`](../by-name/c/checkBackup.md) — 检查备份文件的的完整性和准确性。若所有备份文件均完整且准确，则返回一个空表；否则返回异常的备份文件信息，此时可通过在 `backup` 函数设置 *force* = true 开启强制备份以恢复受损的备份分区数据。
- [`checkpointHaMvcc`](../by-name/c/checkpointHaMvcc.md) — 手动触发指定 MVCC Raft 组的 checkpoint。
- [`clearAllTSDBSymbolBaseCache`](../by-name/c/clearalltsdbsymbolbasecache.md) — 清除缓存中所有未被使用的 symbolBase。其中，未被使用的 symbolBase 是指其对应分区的数据不在 Cache Engine 中，也不在执行的任何事务中。
- [`clearDSCache!`](../by-name/c/clearDSCache_.md) — `clearDSCache!` 函数指示系统在下次执行数据源之后清除缓存。
- [`clearDSCacheNow`](../by-name/c/clearDSCacheNow.md) — 函数 `clearDSCacheNow` 立即清除数据源和缓存。
- [`columnNames`](../by-name/c/columnNames.md) — 返回 *X* 的列名的向量。参见相关函数：[rowNames](../r/rowNames.html)。
- [`createDimensionTable`](../by-name/c/createdimensiontable.md) — 在分布式数据库中创建一个维度表。维度表是分布式数据库中没有进行分区的表，查询时会将表中所有数据加载到内存，适用于存储不频繁更新的小数据集。
- [`createDistributedInMemoryTable`](../by-name/c/createDistributedInMemoryTable.md) — 创建一个分布式共享内存表，按指定分区方案，将数据存储在不同在节点上。该函数只能在数据节点/计算节点执行。
- [`createIPCInMemoryTable`](../by-name/c/createIPCInMemoryTable.md) — 创建一个跨进程共享内存表的句柄。跨进程共享内存表通常作为流计算的输出表。在对时延要求极高的场景下，用户进程通过插件直接访问共享内存获取数据，极大减小了 TCP 等网络传输的时延。
- [`createPartitionedTable`](../by-name/c/createPartitionedTable.md) — 根据 *table* 的结构创建一个空的分区表。
- [`createTable`](../by-name/c/createTable.md)
- [`database`](../by-name/d/database.md) — 创建一个数据库句柄。
- [`disableActivePartition`](../by-name/d/disableActivePartition.md) — 断开与历史数据库的连接。
- [`disableQueryMonitor`](../by-name/d/disableQueryMonitor.md) — 关闭监控查询任务状态的功能。
- [`disableTSDBAsyncSorting`](../by-name/d/disableTSDBAsyncSorting.md) — TSDB 写入 Cache Engine 中的数据会根据 *sortColumns* 排序。写入任务和排序任务可以同步或异步进行，该命令用于关闭异步进行数据排序的功能。只能由管理员在数据节点上执行。
- [`dropColumns!`](../by-name/d/dropColumns_.md) — 从表中删除指定的列。请注意，对于分布式表，不支持删除分区列。
- [`dropDatabase`](../by-name/d/dropDatabase.md) — 删除指定数据库的所有物理文件。
- [`dropDistributedInMemoryTable`](../by-name/d/dropDistributedInMemoryTable.md) — 删除指定的表。该命令只能在数据节点/计算节点上执行。
- [`dropFunctionView`](../by-name/d/dropFunctionView.md) — 删除一个视图或一个命名空间下的所有函数。
- [`dropHaMvccTable`](../by-name/d/dropHaMvccTable.md) — 删除指定名称的高可用 MVCC 表。
- [`dropIPCInMemoryTable`](../by-name/d/dropIPCInMemoryTable.md) — 删除跨进程共享内存表。
- [`dropPartition`](../by-name/d/dropPartition.md) — 删除数据库中指定分区的数据。
- [`dropTable`](../by-name/d/dropTable.md) — 删除指定的表。该命令只能在数据节点/计算节点上执行，不能在控制节点和代理节点上执行。
- [`enableActivePartition`](../by-name/e/enableActivePartition.md) — 创建活动数据库和历史数据库之间的连接。
- [`enableQueryMonitor`](../by-name/e/enableQueryMonitor.md) — 启用监控查询任务状态的功能。系统默认会开启该功能。
- [`enableTSDBAsyncSorting`](../by-name/e/enableTSDBAsyncSorting.md) — TSDB 写入 Cache Engine 中的数据会根据 *sortColumns* 排序。写入任务和排序任务可以同步或异步进行，该命令用于开启异步进行数据排序的功能。异步线程数量由配置参数 *TSDBAsyncSortingWorkerNum* 指定。 只能由管理员在数据节点上执行。请注意，执行该命令前请确认 *TSDBAsyncSortingWorkerNum* 配置值大于0。
- [`existsDatabase`](../by-name/e/existsDatabase.md) — 检查指定数据库是否存在。
- [`existsPartition`](../by-name/e/existsPartition.md) — 检查指定分区是否存在。
- [`existsTable`](../by-name/e/existsTable.md) — 检查指定表是否存在于指定数据库中。
- [`extractTextSchema`](../by-name/e/extractTextSchema.md) — 生成输入数据文件的表的结构。表的结构有两列：列名和数据类型。
- [`flushOLAPCache`](../by-name/f/flushOLAPCache.md) — 将 OLAP 引擎缓冲区里已经完成的事务强制写入数据库。请注意，使用该函数前，需配置 *OLAPCacheEngineSize* 和 *dataSync* = 1。
- [`flushTSDBCache`](../by-name/f/flushTSDBCache.md) — 将 TSDB 引擎缓冲区里已经完成的事务强制写入数据库。请注意，使用该函数前，需配置 *TSDBCacheEngineSize*。
- [`getAllDBGranularity`](../by-name/g/getAllDBGranularity.md) — 该函数只能在数据节点上执行，用于列出该节点上所有数据库的分区粒度。

- [`getBackupMeta`](../by-name/g/getBackupMeta.md) — 查询指定某分布式数据表中某个分区的备份信息。
- [`getBackupStatus`](../by-name/g/getBackupStatus.md) — 查看指定用户的 backup/restore 任务。

- [`getMemLimitOfQueryResult`](../by-name/g/getMemLimitOfQueryResult.md) — 获取单次查询结果占用的内存上限（单位为字节）。

- [`update!`](../by-name/u/update_.md) — 就地更新表中的列。如果 *colNames* 中的某列不存在，将会创建新列。如果指定了过滤条件，只有符合过滤条件的记录行会被更新。

- [`enableTransferCompressionToComputeNode`](../by-name/e/enabletransfercompressiontocomputenode.md) — 用于控制数据节点在向计算节点传输数据前是否进行压缩。
- [`flushComputeNodeMemCache`](../by-name/f/flushcomputenodememcache.md) — 应用于计算组中的计算节点上，将该节点的内存缓存的内容写入磁盘缓存。
- [`getCacheRulesForComputeGroup`](../by-name/g/getcacherulesforcomputegroup.md) — 返回全部或指定计算组中的缓存规则。
- [`getComputeGroupChunksStatus`](../by-name/g/getcomputegroupchunksstatus.md) — 应用于控制节点，获取指定或所有计算组中缓存的所有 chunk （包括 file chunk 和 tablet chunk）的元数据信息。
- [`getComputeNodeCacheDetails`](../by-name/g/getComputeNodeCacheDetails.md) — 在存算分离架构中，在计算组中的计算节点上执行该函数，可返回该节点上的具体缓存信息。
- [`getComputeNodeCacheStat`](../by-name/g/getcomputenodecachestat.md) — 应用于计算组中的计算节点上，返回该节点的二级缓存信息。
- [`getComputeNodeCacheWarmupJobStatus`](../by-name/g/getcomputenodecachewarmupjobstatus.md) — 查询指定 *jobId* 的数据预热任务状态。省略 *jobId* 时返回所有预热任务的状态。
- [`getComputeNodeCachingDelay`](../by-name/g/getcomputenodecachingdelay.md) — 查看当前节点下配置项 *computeNodeCachingDelay* 的生效值，单位为秒。
- [`getPrefetchComputeNodeData`](../by-name/g/getprefetchcomputenodedata.md) — 当前节点下配置项 *enableComputeNodePrefetchData* 的生效值。
- [`isTransferCompressionToComputeNodeEnabled,`](../by-name/i/istransfercompressiontocomputenodeenabled.md) — 查询数据节点到计算节点的数据传输是否启用压缩。
- [`removeCacheRulesForComputeGroup`](../by-name/r/removecacherulesforcomputegroup.md) — 移除计算组的缓存规则，指定需要取消计算组缓存的表。
- [`setComputeNodeCachingDelay`](../by-name/s/setcomputenodecachingdelay.md) — 在线设置当前节点下配置项 *computeNodeCachingDelay* 的生效值。只能由管理员在控制节点执行。
- [`setPrefetchComputeNodeData`](../by-name/s/setprefetchcomputenodedata.md) — 在线设置当前节点下配置项 *enableComputeNodePrefetchData* 的生效值。只能由管理员在控制节点执行。
- [`warmupComputeNodeCache`](../by-name/w/warmupcomputenodecache.md) — 创建数据预热任务，将指定数据缓存至计算组。

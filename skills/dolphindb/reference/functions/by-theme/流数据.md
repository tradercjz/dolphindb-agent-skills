# 流数据

## 流表操作

- [`appendForJoin`](../by-name/a/appendForJoin.md) — 将数据插入流数据 join 引擎中。
- [`appendMsg`](../by-name/a/appendMsg.md) — 当流数据引擎启用快照机制（snapshot）且未开启 RaftGroup 时，订阅函数 [subscribeTable](../s/subscribeTable.html) 的handler参数必须为 `appendMsg` 函数，将数据写入流数据引擎。
- [`clearTablePersistence`](../by-name/c/clearTablePersistence.md) — 停止将流数据表持久化到磁盘，然后删除磁盘上表的内容，但仍然保留表的结构。
- [`disableTablePersistence`](../by-name/d/disableTablePersistence.md) — 停止把表持久化到磁盘，后续表更新的内容将不会持久化到磁盘上。
- [`dropStreamTable`](../by-name/d/dropStreamTable.md) — 删除流数据表。如果流数据表启用了持久化，该函数也会将磁盘上持久化的数据删除。
- [`enableTableCachePurge`](../by-name/e/enabletablecachepurge.md) — 为非持久化流表开启自动清理缓存。
- [`enableTablePersistence`](../by-name/e/enableTablePersistence.md) — 该命令将共享的流计算表保存到磁盘上。
- [`enableTableShareAndCachePurge`](../by-name/e/enabletableshareandcachepurge.md) — 将非持久化的流数据表共享，并设置定时清理。
- [`enableTableShareAndPersistence`](../by-name/e/enableTableShareAndPersistence.md) — 该命令将流数据表共享，并把它持久化到磁盘上。
- [`existsStreamTable`](../by-name/e/existsStreamTable.md) — 查询指定的流数据表是否存在。
- [`existsSubscriptionTopic`](../by-name/e/existsSubscriptionTopic.md) — 查询共享流数据表的订阅主题是否存在。存在返回 true，不存在则返回 false。
- [`getStreamTables`](../by-name/g/getstreamtables.md) — 获取流数据表的信息。
- [`haStreamTable`](../by-name/h/haStreamTable.md) — 创建高可用流数据表。该函数只能在启用流数据高可用后使用。要启用流数据高可用，用户需要在集群配置文件 cluster.cfg 中指定配置参数 *streamingHAMode* 和 *streamingRaftGroups*。系统启动时，配置参数 *streamingRaftGroup* 指定的数据节点/计算节点组成 Raft 组，一个数据节点/计算节点作为 Leader，其他数据节点/计算节点作为 Follower。…
- [`keyedStreamTable`](../by-name/k/keyedStreamTable.md) — 创建键值流数据表。该表的主键不允许包含重复值。主键支持一个或多个字段。
- [`removeTopicOffset`](../by-name/r/removeTopicOffset.md) — 删除给定订阅主题（topic）的持久化保存的最新一条已经处理订阅数据的偏移量（在 [subscribeTable](../s/subscribeTable.html) 函数中通过指定 *persistOffset* 参数为 true 获得）。
- [`setStreamTableFilterColumn`](../by-name/s/setStreamTableFilterColumn.md) — 指定流数据表的过滤列。该函数需要配合 [subscribeTable](subscribeTable.html) 函数的 *filter* 参数一起使用。*filter* 是一个向量，*streamTable* 的 *columnName* 列在 *filter* 中的数据才会发布到订阅端，不在 *filter* 中的数据不会发布。一个流数据表只能指定一个过滤列。
- [`setStreamTableTimestamp`](../by-name/s/setstreamtabletimestamp.md) — 为指定的流数据表设置一个时间戳列（支持所有时间类型）。设置后，每次向流表写入数据时，系统会记录写入时的系统时间到该时间戳列。这有助于用户统计实时数据写入流表时的延迟。
- `share`
- [`subscribeTable`](../by-name/s/subscribeTable.md) — 从客户端节点订阅本地或远程服务器的流数据表。可在 *handler* 调用函数来处理订阅数据。
- [`unsubscribeTable`](../by-name/u/unsubscribeTable.md) — 在信息订阅端的节点执行，以停止向信息发布者订阅数据。在调用该命令时，会删除流计算执行线程的队列中未处理的消息。
- [`streamTable`](../by-name/s/streamTable.md) — 创建[流数据表](../../stream/str_table.html)。流数据表可处理并发的读写操作。

## 计算引擎

- [`createAnomalyDetectionEngine`](../by-name/c/createAnomalyDetectionEngine.md) — 创建异常检测引擎，返回一个表对象，向该表写入数据意味着将数据注入异常检测引擎进行计算。异常检测引擎会根据异常指标，对每条记录进行检测。
- [`createAsofJoinEngine`](../by-name/c/createAsofJoinEngine.md) — 创建流数据 asof join 引擎。返回一个左、右两表 asof join 后的表对象。注入 asof join 引擎的左、右两表将以 `matchingColumn` + `timeColumn` (或系统时间) 作为连接列，在右表中选取与连接列匹配的，在给定的左表时间戳前且最接近的记录。该引擎适用于两个数据源没有完全匹配的记录，需要按时间段作连接，以获取最新信息的场景。
- [`createCrossSectionalEngine`](../by-name/c/createCrossSectionalEngine.md) — 创建横截面引擎，返回一个以 *keyColumn* 作为键值的键值表。
- [`createDailyTimeSeriesEngine`](../by-name/c/createDailyTimeSeriesEngine.md) — 创建流数据日级时间序列引擎。日级时间序列引擎和时间序列引擎窗口划分和计算规则基本一致，但在此基础上做了如下拓展：
- [`createDualOwnershipReactiveStateEngine`](../by-name/c/createDualOwnershipReactiveStateEngine.md) — Dual Ownership Reactive State Engine 是对响应式状态引擎的扩展，支持对同一张流数据表指定两种不同的分组方式分别应用不同的指标进行并行计算。与响应式状态级联实现相比，该函数能极大地提升计算性能。
- [`createEquiJoinEngine`](../by-name/c/createEquiJoinEngine.md) — 创建流数据等值连接引擎。返回一个左、右两表 equi join 后的表对象。注入等值连接引擎的左、右两表将以 *matchingColumn* + *timeColumn* 作为连接列，返回两表中连接字段相等的行。
- [`createEqualJoinEngine`](../by-name/c/createEqualJoinEngine.md)
- [`createLeftSemiJoinEngine`](../by-name/c/createLeftSemiJoinEngine.md) — 创建流数据左半等值连接引擎，返回一个左、右表关联后的表对象。对于左表每一条数据，都去匹配右表相同 *matchingColumn* 的数据，若无匹配的右表记录，则不输出。若匹配多条右表记录，则由 *updateRightTable* 参数决定连接右表的第一条记录还是最新一条记录。
- [`createLookupJoinEngine`](../by-name/c/createLookupJoinEngine.md) — 创建流数据表的 lookup join 引擎，该引擎以 *matchingColumn*作为连接列，将左表（必须为流数据表）与右表进行左连接或内连接。lookup join 引擎常用于右表更新不频繁的场景（如保存了日频指标的维度表）。若右表为非流数据表，则需设置定时刷新该表数据。
- [`createNarrowReactiveStateEngine`](../by-name/c/createnarrowreactivestateengine.md) — 创建生成窄表的响应式状态引擎。其计算规则和计算方式与 createReactiveStateEngine 一致，输出形式不同。createReactiveStateEngine 将每次计算的指标输出为一行的不同列，而该引擎则把通过 metricNames 指定指标的计算以窄表的形式输出，即计算结果输出到不同行的相同列。
- [`createOrderBookSnapshotEngine`](../by-name/c/createorderbooksnapshotengine.md) — `createOrderBookSnapshotEngine` 用于创建一个订单簿快照引擎（OrderBook Snapshot Engine）。该引擎根据逐笔委托和逐笔成交数据实时合成指定频率（如 1 秒、100ms、30ms 等）的订单簿快照数据。
- [`createReactiveStateEngine`](../by-name/c/createReactiveStateEngine.md) — 创建响应式状态引擎。返回一个表对象，向该表写入数据意味着这些数据进入响应式状态引擎进行计算。
- [`createRuleEngine`](../by-name/c/createRuleEngine.md) — `createRuleEngine` 用于创建一个规则引擎。
- [`createSessionWindowEngine`](../by-name/c/createSessionWindowEngine.md) — 创建流数据会话窗口引擎。
- [`createSnapshotJoinEngine`](../by-name/c/createsnapshotjoinengine.md) — 创建流数据表的快照连接引擎。该引擎以*matchingColumn*作为连接列，将两个流数据表进行实时的内连接（等值连接）或外连接（左/右连接）。
- [`createSparseReactiveStateEngine`](../by-name/c/createSparseReactiveStateEngine.md) — 创建一个稀疏响应式状态引擎，用于对数据进行稀疏状态计算：仅当某些特定的数据到达时，才触发与之相关的状态计算。
- [`createStreamDispatchEngine`](../by-name/c/createStreamDispatchEngine.md) — 创建流数据分发引擎，返回一个表对象。该引擎将输入的数据分发到不同的输出表，以实现负载均衡。其中输出表可以是内存表，分布式表或流数据引擎。
- [`createTimeBucketEngine`](../by-name/c/createtimebucketengine.md) — 创建一个自定义窗口长度（长度相同或不同）的时间序列聚合引擎。该引擎基于数据时间，按照指定的长度对输入数据进行窗口划分，并在窗口内执行增量聚合计算。通常和时序聚合引擎（createTimeSeriesEngine ）级联使用，接收其输出秒级或分钟级聚合结果（例如 K 线）进行进一步处理。
- [`createTimeSeriesEngine`](../by-name/c/createTimeSeriesEngine.md) — 创建流数据时间序列引擎，以实现基于时间的滑动窗口或滚动窗口进行实时计算。
- [`createWindowJoinEngine`](../by-name/c/createWindowJoinEngine.md) — 创建流数据 window join 引擎。返回一个左、右两表实时 [window join](../../progr/sql/windowjoin.html) 后的表对象。
- [`streamEngineParser`](../by-name/s/streamEngineParser.md) — 在实际场景下，用户的因子可能嵌套横截面、历史状态、时序窗口、表连接等多种复杂逻辑，需要多个计算引擎的级联才能实现。`streamEngineParser` 为只涉及横截面、历史状态、时序窗口三种逻辑嵌套的复杂因子，提供了一个统一的计算入口。用户无需编写复杂的级联代码，只需将嵌套因子按规则改写后传入`streamEngineParser` ，系统将自动解析各层嵌套涉及的计算逻辑，形成流计算引擎流水线，自动分发给对应的引擎进行计算。
- [`createCryptoOrderBookEngine`](../by-name/c/createcryptoorderbookengine.md) — 根据数字货币全量订单簿快照和增量订单簿更新数据，实时更新数字货币交易对的订单簿，并输出指定深度的订单簿信息。

## 工具函数

- [`addMetrics`](../by-name/a/addMetrics.md) — 动态增加流数据引擎的计算指标。
- [`addReactiveMetrics`](../by-name/a/addreactivemetrics.md) — 动态增加 narrowReactiveStateEngine 的计算指标。更新后的计算指标在引擎后续注入的数据时生效。
- [`addSparseReactiveMetrics`](../by-name/a/addSparseReactiveMetrics.md) — 为指定的 SparseReactiveStateEngine 增加稀疏状态计算规则。
- [`addStatelessMetrics`](../by-name/a/addstatelessmetrics.md) — 向响应式无状态引擎中增加计算逻辑。
- [`conditionalIterate`](../by-name/c/conditionalIterate.md) — 通过条件迭代实现因子中的递归逻辑。
- [`deleteSparseReactiveMetric`](../by-name/d/deleteSparseReactiveMetric.md) — 删除指定 SparseReactiveStateEngine 中 `outputMetricKey` 对应的规则：若存在则删除并返回 true，否则返回 false。
- [`dropStreamEngine`](../by-name/d/dropStreamEngine.md) — 释放指定的流计算引擎的定义。
- [`dropAggregator`](../by-name/d/dropAggregator.md)
- [`getComputeNodeCacheWarmupJobStatus`](../by-name/g/getcomputenodecachewarmupjobstatus.md) — 查询指定 *jobId* 的数据预热任务状态。省略 *jobId* 时返回所有预热任务的状态。
- [`forceTriggerOrderBookSnapshot`](../by-name/f/forceTriggerOrderBookSnapshot.md) — 该函数用于强制触发订单簿快照输出，适用于不满足常规触发条件（如时间窗口或数据）但仍需生成快照的场景。调用后，快照会立即生成并输出。
- [`getLeftStream`](../by-name/g/getLeftStream.md) — 返回连接引擎左表的表结构对象。向该对象注入的数据，会注入到 *joinEngine* 中。
- [`getRightStream`](../by-name/g/getRightStream.md) — 返回连接引擎右表的表结构对象。向该对象注入的数据，会注入到 *joinEngine* 中。
- [`getPersistenceMeta`](../by-name/g/getPersistenceMeta.md) — 查询启用了持久化的共享流数据表的元数据。
- [`getReactiveMetrics`](../by-name/g/getreactivemetrics.md) — 获取指定名称的 narrowReactiveStateEngine 的计算指标列。
- [`getSnapshotMsgId`](../by-name/g/getSnapshotMsgId.md) — 用于订阅断开后重新订阅时，获取指定流数据引擎的最近一个快照（snapshot）的 msgId。启用 snapshot 后，重订阅时，`subscribeTable` 函数的 *offset* 参数设置为 getSnapshotMsgId(engine)+1，引擎会加载 snapshot，并从 getSnapshotMsgId(engine) 之后一条消息开始重新订阅。
- [`getSparseReactiveMetrics`](../by-name/g/getSparseReactiveMetrics.md) — 获取指定稀疏响应式状态引擎当前配置的计算规则。
- [`getStatelessMetrics`](../by-name/g/getstatelessmetrics.md) — 获取响应式无状态引擎的计算逻辑。
- [`getStreamEngine`](../by-name/g/getStreamEngine.md) — 返回流数据引擎的句柄，可以作为 [subscribeTable](../s/subscribeTable.html) 函数的 *handler* 参数。
- [`getAggregator`](../by-name/g/getAggregator.md)
- [`getStreamEngineList`](../by-name/g/getstreamenginelist.md) — 获取当前节点上所有流计算引擎的信息。
- [`getStreamEngineStat`](../by-name/g/getStreamEngineStat.md) — 查询流数据引擎的状态。
- [`getAggregatorStat`](../by-name/g/getAggregatorStat.md)
- [`getRules`](../by-name/g/getrules.md) — 查询规则引擎的规则信息，包括规则集、规则检查策略和回调函数等。
- [`getStreamingLeader`](../by-name/g/getStreamingLeader.md) — 获取流数据 Raft 组中的 Leader。
- [`getStreamingRaftGroups`](../by-name/g/getStreamingRaftGroups.md) — 获取当前节点所在的流数据 Raft 组的信息。
- [`getStreamingStat`](../by-name/g/getStreamingStat.md) — 监控流计算任务的运行状态。
- [`getStreamTableCacheOffset`](../by-name/g/getstreamtablecacheoffset.md) — 查看指定的流表在内存中的最旧的记录相较于流表已写入的总记录数的偏移量（offset），即总记录数减去内存中数据行的值。
- [`getStreamTableFilterColumn`](../by-name/g/getStreamTableFilterColumn.md) — 返回由函数 [setStreamTableFilterColumn](../s/setStreamTableFilterColumn.html) 指定的流数据表中过滤列的名称。
- [`getSubscriptionTopic`](../by-name/g/getSubscriptionTopic.md) — 查询订阅主题的名称和流数据表的列名。
- [`getTopicProcessedOffset`](../by-name/g/getTopicProcessedOffset.md) — 如果 `subscribeTable` 函数的 *persistOffset* 参数为 true，那么该函数返回最新一条已经处理的订阅数据的偏移量；如果 `subscribeTable` 函数的 *persistOffset* 参数为 false，那么该函数返回-1。
- [`stateIterate`](../by-name/s/stateIterate.md) — 通过线性迭代实现线性递归。
- [`warmupComputeNodeCache`](../by-name/w/warmupcomputenodecache.md) — 创建数据预热任务，将指定数据缓存至计算组。
- [`warmupStreamEngine`](../by-name/w/warmupStreamEngine.md) — 把数据写入流数据引擎，但是不输出结果。下一批次数据写入此流数据引擎，可以利用已计算的结果来加速计算。

## 流式 SQL

- [`declareStreamingSQLTable`](../by-name/d/declareStreamingSQLTable.md) — 将指定表声明为流式 SQL 的输入表。仅被声明的表才能通过 `registerStreamingSQL` 注册流式 SQL 查询。该声明操作不会影响该表在普通 SQL 查询或其他功能中的使用。
- [`getStreamingSQLStatus`](../by-name/g/getStreamingSQLStatus.md) — 查看流式 SQL 查询的运行状态。
- [`listStreamingSQLTables`](../by-name/l/listStreamingSQLTables.md) — 列出当前用户已通过 `declareStreamingSQLTable` 声明的所有流式 SQL 表。
- [`registerStreamingSQL`](../by-name/r/registerStreamingSQL.md) — 注册一条流式 SQL 查询，成功注册后返回一个唯一的 *queryId*。系统会基于该查询生成一个与 *queryId* 同名的共享流表，用于写入结果变更日志。
- [`revokeStreamingSQL`](../by-name/r/revokeStreamingSQL.md) — 注销一个通过 `registerStreamingSQL` 注册的流式 SQL 查询。只能注销当前用户自己注册的查询。
- [`revokeStreamingSQLTable`](../by-name/r/revokeStreamingSQLTable.md) — 注销一个通过 `declareStreamingSQLTable` 声明的流式 SQL 表。
- [`subscribeStreamingSQL`](../by-name/s/subscribeStreamingSQL.md) — 订阅指定的流式 SQL 查询结果。订阅端会接收增量日志，并基于这些日志维护一个实时更新的共享结果表，保证查询结果随数据变更持续刷新。
- [`unsubscribeStreamingSQL`](../by-name/u/unsubscribeStreamingSQL.md) — 取消订阅指定的流式 SQL 查询结果。取消订阅后，订阅端的实时结果表将停止更新。
- [`getStreamingSQLSubscriptionInfo`](../by-name/g/getstreamingsqlsubscriptioninfo.md) — 查询指定流式 SQL 的元数据信息。
